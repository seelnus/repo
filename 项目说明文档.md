# 内部问卷系统 · 项目说明文档

**文档版本**：v1.0  
**更新日期**：2026-06-30  
**项目地址**：https://hr.mmcb.top  

---

## 一、项目概述

内部问卷系统是面向企业内部员工的在线问卷平台，支持管理员创建和管理多类型问卷，员工通过企业微信身份认证后完成填写。系统提供完整的问卷生命周期管理，包括创建、发布、数据收集、结果导出等核心功能，并支持白名单访问控制，确保问卷定向触达指定人员。

## 三、系统架构与部署

### 部署环境
- **服务器**：阿里云 ECS（4核 / 7.8GB 内存）
- **操作系统**：Linux（宝塔面板管理）
- **域名**：hr.mmcb.top（HTTPS）
- **反向代理**：宝塔 Nginx

### 容器化部署（Docker Compose）

系统通过 Docker Compose 编排，共运行 4 个容器：

| 容器名 | 镜像 | 端口 | 说明 |
|--------|------|------|------|
| survey-app-app-frontend-1 | node:20-alpine | 5174 | 前端 Vite 开发服务器 |
| survey-app-app-backend-1 | node:20-alpine | 3100 | 后端 NestJS API 服务 |
| survey-app-mysql-1 | mysql:8.0 | 3306 | MySQL 数据库 |
| survey-app-minio-1 | minio/minio | 9000/9001 | 文件对象存储 |

### Nginx 路由规则
```
https://hr.mmcb.top/api/*  →  后端 API（端口 3100）
https://hr.mmcb.top/*      →  前端页面（端口 5174）
```

### 数据库
- 数据库名：`survey_app`
- 数据持久化通过 Docker Volume 挂载，容器重启不丢失数据
- 每次后端启动时自动执行 `prisma migrate deploy`，完成数据库结构升级

### 代码管理
- 代码托管于 GitHub，服务器通过 `git pull` + `docker compose build` 完成更新部署
- 源码以 Volume 形式挂载进容器，后端支持热重载

## 四、功能说明

### 4.1 管理员端（后台）

#### 登录与权限
- 管理员通过手机号 + 密码登录
- 支持主管理员与普通管理员角色区分
- JWT Token 认证，登录态保存于浏览器 localStorage

#### 问卷管理
- **文件夹系统**：所有问卷按文件夹归类管理，支持新建、重命名、删除文件夹；未归类问卷统一放入"未分类"
- **创建问卷**：支持拖拽式题目编辑器，填写问卷标题、类型、题目配置
- **问卷类型**：案例收集、测评问卷、宣传文件（三种）
- **发布/禁用**：问卷可在「草稿 → 发布 → 禁用」状态间切换
- **分享链接**：每份问卷有唯一分享链接，支持企微扫码访问
- **移动问卷**：支持将问卷在不同文件夹之间移动
- **删除问卷**：逻辑删除，数据不物理清除

#### 题目类型支持
| 题型 | 说明 |
|------|------|
| 单选题 | 支持"其他"选项 |
| 多选题 | 支持"其他"选项 |
| 评分题 | 自定义最高分 |
| 文字描述 | 纯展示内容块 |
| 单行文本 | 简短文字输入 |
| 多行文本 | 长文字输入 |
| 文件上传 | 限制格式与大小 |
| 日期 | 日期选择 |
| 日期时间 | 日期+时间选择 |
| 题目联动 | 根据上一题答案控制题目显示/隐藏 |

#### 数据查看与导出
- **查看答卷**：按问卷查看所有提交记录，支持评分和评语
- **导出 CSV**：支持按填写时间范围筛选导出，字段包含填写人、提交时间、各题答案

#### 白名单管理
- 对问卷设置访问白名单，仅白名单内联系人可填写
- 白名单支持启用/禁用开关
- 联系人可批量导入

#### 联系人管理
- 维护公司员工通讯录，字段包含：姓名、部门、工号、职位、手机号、邮箱、标签
- 联系人可关联白名单，作为问卷访问控制依据

---

### 4.2 员工端（问卷填写）

- **入口**：通过管理员分享的链接或企微消息进入
- **身份认证**：企业微信 OAuth 扫码授权，获取企微身份后方可填写
- **填写限制**：启用白名单的问卷，非名单内员工访问提示无权限
- **提交**：提交后不可修改（每人限填一次）
- **宣传文件**：支持展示图文内容，无需填写直接阅读

## 五、操作说明

### 5.1 管理员登录

1. 打开 https://hr.mmcb.top，进入登录页
2. 输入管理员手机号和密码，点击「登录」
3. 登录成功后进入后台管理首页

> 初始管理员账号由技术人员在数据库初始化时创建。如需新增管理员，由主管理员在「管理员管理」页面操作。

---

### 5.2 联系人管理

联系人是白名单的基础数据，建议在创建问卷白名单前先维护好联系人。

**添加联系人**
1. 进入左侧菜单「联系人管理」
2. 点击「新增联系人」
3. 填写姓名（必填）、手机号（必填）、部门、工号、职位、邮箱、标签等信息
4. 点击「保存」

**批量导入联系人**
1. 进入「联系人管理」页面
2. 点击「批量导入」，按照模板格式准备 Excel/CSV 文件
3. 上传文件后系统自动解析并导入

**编辑 / 删除联系人**
- 在联系人列表找到对应记录，点击「编辑」修改信息
- 点击「删除」可移除该联系人（已关联白名单的联系人删除后自动从白名单中移除）

---

### 5.3 创建问卷

1. 进入左侧菜单「问卷管理」
2. 选择或创建目标文件夹（见 5.4），点击进入文件夹
3. 点击右上角「新建问卷」
4. 填写基本信息：
   - **问卷名称**：唯一标识，不可重复
   - **问卷类型**：选择「案例收集」「测评问卷」或「宣传文件」
5. 进入题目编辑器，点击「添加题目」逐题配置：
   - 选择题型（单选、多选、评分、文本、文件上传等）
   - 填写题目标题和选项
   - 勾选「必填」控制是否必答
   - 如需题目联动，在「显示条件」中设置依赖题目及触发值
6. 点击「保存」，问卷以「草稿」状态保存

> 宣传文件类型的问卷支持在顶部添加富文本图文内容（contentHtml），用于展示通知、公告等内容，员工无需填写即可阅读。

---

### 5.4 文件夹管理

文件夹用于对问卷分类归档，所有问卷必须归属于某个文件夹。

**新建文件夹**
1. 进入「问卷管理」，点击「新建文件夹」
2. 输入文件夹名称，点击「保存」

**重命名文件夹**
1. 在文件夹卡片右上角点击编辑图标（铅笔）
2. 修改名称后点击「保存」

**删除文件夹**
1. 点击文件夹卡片右上角的删除图标
2. 确认弹窗提示——文件夹内问卷会自动移入「未分类」，问卷本身不会被删除

**移动问卷到其他文件夹**
1. 进入文件夹后，在问卷列表找到目标问卷
2. 点击操作栏「移动」
3. 选择目标文件夹（或「未分类」），点击「确认移动」

---

### 5.5 发布问卷

问卷创建后默认为「草稿」状态，不对外开放，需手动发布。

1. 进入文件夹，找到目标问卷
2. 将列表中「启用状态」列的开关拨到「已启用」
3. 问卷状态变为「已发布」，员工可通过分享链接访问填写

**禁用问卷**
- 将启用开关拨回关闭，问卷立即停止接受填写，已提交的数据不受影响

---

### 5.6 分享问卷

1. 进入文件夹，点击问卷右侧「分享」按钮
2. 页面展示该问卷的专属链接和二维码
3. 将链接或二维码发送给填写人员（可通过企微消息、群公告等方式）

> 员工打开链接后需先完成企业微信 OAuth 扫码认证，认证通过后进入填写页面。

---

### 5.7 白名单管理

白名单用于限制问卷仅向指定人员开放。未设置白名单的问卷，所有完成企微认证的员工均可填写。

**为问卷创建白名单**
1. 进入左侧菜单「白名单管理」
2. 点击「新建白名单」，选择要绑定的问卷
3. 在成员列表中搜索并添加联系人
4. 确认保存后，白名单默认为「启用」状态

**启用 / 禁用白名单**
- 在白名单列表中，通过开关控制白名单生效状态
- 禁用后该问卷恢复为所有人可填写；重新启用后恢复限制

**添加 / 移除成员**
1. 点击白名单右侧「管理成员」
2. 搜索联系人姓名或手机号，勾选后点击「添加」
3. 移除成员点击成员列表中的「删除」图标

> 白名单成员必须已存在于联系人列表，请先在「联系人管理」中维护人员信息。

---

### 5.8 查看答卷数据

1. 进入文件夹，点击目标问卷右侧「数据」按钮
2. 页面展示所有已提交的答卷列表，包含：
   - 提交人企微姓名
   - 提交时间
   - 各题答案详情
3. 点击单条答卷可展开查看完整内容
4. 管理员可对每条答卷添加「评分」和「评语」，用于内部记录

---

### 5.9 导出 CSV

1. 进入文件夹，点击目标问卷右侧「导出 CSV」按钮
2. 弹窗中可选择填写时间范围（开始日期 / 结束日期）
   - 不选时间范围则导出该问卷所有历史数据
3. 点击「导出」，浏览器自动下载 CSV 文件

导出字段包含：提交记录 ID、填写人企微账号、填写人姓名、提交时间、各题目答案（按列展开）。

---

### 5.10 员工填写问卷

1. 员工收到分享链接后，用手机或电脑打开
2. 页面跳转至企业微信授权，点击「同意授权」完成身份认证
3. 认证成功后进入问卷填写页，按题目顺序作答
4. 所有必填题完成后，点击「提交」完成填写
5. 提交成功后显示感谢页，问卷不可重复提交

> 若员工不在白名单内，打开链接后会提示「该问卷暂未开放」，无法进入填写页。

## 六、数据库主要数据结构

| 表名 | 说明 |
|------|------|
| admin_users | 管理员账号 |
| survey_folders | 问卷文件夹 |
| surveys | 问卷（含题目 JSON） |
| survey_responses | 问卷填写记录 |
| response_comments | 管理员对答卷的评分/评语 |
| survey_whitelists | 问卷白名单配置 |
| whitelist_members | 白名单成员 |
| contacts | 联系人通讯录 |
| wecom_users | 企微用户身份缓存 |

## 八、运维说明

### 更新部署流程
```bash
cd /www/wwwroot/survey-app
git pull
docker compose build app-backend --no-cache
docker compose up -d app-backend
```

### 查看日志
```bash
# 后端日志
docker logs survey-app-app-backend-1 -f

# 前端日志
docker logs survey-app-app-frontend-1 -f
```

### 进入数据库
```bash
docker exec -it survey-app-mysql-1 mysql -u survey -psurvey_password survey_app
```

### 备份数据库
```bash
docker exec survey-app-mysql-1 mysqldump -u survey -psurvey_password survey_app > backup_$(date +%Y%m%d).sql
```
