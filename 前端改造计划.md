# 前端改造计划（填鸭风格）

> 参考设计原型：`设计原型-填鸭风格.html`  
> 目标文件：`frontend/src/index.css`、`frontend/src/App.tsx`  
> 策略：保留 Ant Design 组件树，以 CSS 覆盖 + 最小 JSX 改动实现视觉升级

---

## 阶段一：CSS Token 系统 & 全局基础样式

**文件：** `index.css`

### 改动内容
- 新增 `:root` CSS 变量（设计令牌）：
  ```
  --primary: #4E73F5           // 主蓝色
  --primary-end: #7C54E8       // 渐变终点紫色
  --sidebar-bg: #1B2A4A        // 侧边栏深蓝
  --sidebar-text: rgba(255,255,255,0.75)
  --bg: #F0F2F5                // 页面背景
  --surface: #ffffff           // 卡片白
  --border: #E8ECF2
  --text-primary: #1F2937
  --text-secondary: #6B7280
  --radius: 12px
  --shadow-card: 0 2px 12px rgba(0,0,0,0.06)
  ```
- `body` 背景改为 `var(--bg)`，字色改为 `var(--text-primary)`
- Ant Design ConfigProvider `token.colorPrimary` 改为 `#4E73F5`（需在 `App.tsx` 中更新 `<ConfigProvider>` props）

**不涉及 JSX 改动**（除 ConfigProvider token）

---

## 阶段二：登录页

**文件：** `App.tsx`（LoginPage 函数）、`index.css`（`.login-page` 等）

### 现状
居中单卡片，普通白底。

### 目标
左右分栏：
- **左侧（flex: 1）**：蓝紫渐变背景，品牌名 + 副标题 + 3 条系统亮点文字
- **右侧（460px）**：深蓝 `#1B2A4A` 背景，白色登录卡，渐变提交按钮

### JSX 改动
```
LoginPage: <div className="login-page">
  → 内部新增 .login-left（品牌文案）和 .login-right（表单卡片）两个子 div
```

### CSS 改动
- `.login-page`：改为 `display: flex; min-height: 100vh;`
- 新增 `.login-left`、`.login-right`、`.login-card` 样式
- 登录按钮用渐变背景（`linear-gradient(135deg, #4E73F5, #7C54E8)`）

## 阶段三：管理后台 Shell & 侧边栏

**文件：** `App.tsx`（AdminShell 函数）、`index.css`

### 现状
`<Layout.Sider theme="light">`，白底，无 Logo 区域

### 目标
深蓝侧边栏 + 图标 + 选中高亮渐变

### JSX 改动（最小）
```
<Layout.Sider theme="dark">   // light → dark
  Logo div 改样式
```
用 `<ConfigProvider>` 的 `components.Menu` token 或 CSS 覆盖 `.ant-menu-dark` 配色

### CSS 改动
- `.app-shell .ant-layout-sider`：背景改为 `var(--sidebar-bg)`
- `.ant-menu-dark`：背景透明，文字色 `var(--sidebar-text)`
- `.ant-menu-dark .ant-menu-item-selected`：渐变背景，圆角 8px
- Header 区域：白底改为更简洁的布局

## 阶段四：问卷管理列表

**文件：** `App.tsx`（SurveyList 函数）、`index.css`

### 现状
一张白色 Card 包裹 Table，顶部工具栏

### 目标
保留 Table（数据结构已完善，不改为卡片网格），但做以下升级：
- 顶部添加 4 个数据统计小卡片（仅静态展示，从列表数据动态计算）：
  - 问卷总数 / 已启用 / 已禁用 / 本月新增
- 表格行 hover 背景色改为 `#EEF2FF`，选中行已有样式 `.response-row-active`
- 新建按钮用渐变背景

### JSX 改动
- SurveyList 返回值顶部插入统计卡片行（`.stats-row`，4 个 `.stat-card`）
- 数据从已有 `data` state 计算，无需新增 API

### CSS 改动
- 新增 `.stats-row`、`.stat-card`、`.stat-card-icon`、`.stat-card-value` 等样式

## 阶段六：用户填写页（FillPage）

**文件：** `App.tsx`（FillPage 函数及其子组件）、`index.css`

### 现状
居中单列，最大宽度 480px，顶部无特殊设计

### 目标（TDuck 最标志性的设计）
- **渐变大 Banner 头图**：包含问卷标题和描述，高约 180px
- **悬浮进度卡片**：显示「已完成 X / 共 N 题」+ 渐变进度条，粘性定位
- **题目卡片**：白色圆角卡，已答题卡左边框或底部加绿色标记
- **提交按钮**：渐变背景

### JSX 改动（填写页改动最多）
- FillPage 整体结构：
  ```
  div.fill-page
    div.fill-banner（新增渐变头图）
    div.fill-body
      div.fill-progress-card（新增进度卡片）
      [题目卡片列表]
      提交按钮
  ```
- 进度逻辑：已有 `answers` state，用它计算已填数 / 总题数，实时更新进度条宽度

### CSS 改动
- `.fill-banner`：渐变背景 + 居中文字
- `.fill-progress-card`：白卡，`position: sticky; top: 16px;`，渐变进度条
- `.fill-question-card.answered`：`border-left: 3px solid #52C41A`

## 执行节奏

```
阶段一 → 阶段二 → 阶段三 → [构建验证] → 阶段四 → 阶段五 → 阶段六 → [最终验证]
```

每个阶段完成后独立可运行，出现问题可定点回退。  
**不改动后端接口、路由结构、Ant Design 组件层级，仅覆盖样式 + 最小 JSX 插入。**

---

## 风险说明

- Ant Design CSS-in-JS 的 `:where()` 选择器权重为 0，自定义 CSS 可直接覆盖，但部分组件内部样式需加 `.ant-xxx` 前缀
- 填写页（阶段六）改动最多，如果 FillPage 函数超过 400 行，建议拆分子组件后再修改
- 侧边栏 `theme="dark"` 切换后，Ant Design 会自动处理 Menu 文字色，但需要验证与自定义背景的叠加效果
