## 二、新功能：360 环评（核心）

### 数据准备
- [ ] 「联系人」里有 ≥4 个**同一部门**（如"技术组"）的联系人
- [ ] 「问卷管理」里有 2 份问卷当模板（自评问卷 / 他评问卷，各至少 1 道题）

### 批次与配置
- [ ] 新建批次 → 能进入详情页
- [ ] 「① 配置」填写参评范围（部门）+ 绑定自评/他评模板 → 保存成功；刷新后配置仍在
- [ ] 编辑批次名称/状态、删除批次 均正常

### 🔴 生成关系（监工点 2）
- [ ] 4 个同部门普通员工 → 生成报告：普通 4、自评 4、他评 12、**总数 16**
- [ ] 报告底部校验行数字自洽：`4 人 ⇒ 4 份自评 + 12 份他评 = 16 条`
- [ ] 再次点「生成」，人工配置（manual）的关系不会被冲掉（见"人工配置"一节）

### 🔴 领导排除
- [ ] 到「联系人」给其中 1 人的**标签(tags)加上"领导"**二字 → 回批次重新生成
- [ ] 报告变为：领导 1、普通 3、**总数 9**；该领导不出现在任何自动生成的关系中

### 🔴 复核列表（监工点 3）
- [ ] 领导那一行标红「领导待配」；普通员工显示「完整」
- [ ] 顶部统计（总数 / 已完整 / 异常）数字正确
- [ ] 「仅看异常」开关能只显示异常行
- [ ] 顶部异常类型标签（领导待配 / 双漏 / 未配自评 / 无人评价）数量正确

### 人工配置（监工点 5 · 上）
- [ ] 「④ 关系明细」→「人工添加关系」，选评价人 + 被评人 + 类型 → 添加成功，来源显示「人工」
- [ ] 重复添加同一对（评价人+被评人）→ 提示"已存在"
- [ ] 删除一条**未填写**的关系 → 成功；**已填写**的关系删除按钮为禁用

### 🔴 填写闭环（监工点 4）
- [ ] 打开 http://localhost:5174/eval-fill ，以「杨思豪」身份进入 → 看到 1 份自评 + 2 份他评
- [ ] 填一份并提交 → 该份显示「已完成」
- [ ] 同一份再进去 → 提示已提交、不能重复填
- [ ] 填写端**只显示"我要评谁"，看不到"别人怎么评我"**（他评结果对被评人不可见）
- [ ] 回后台「复核列表」刷新 → 对应人的"被几人评 / 评几人"计数 +1

### 导出（监工点 5 · 下）
- [ ] 批次详情「导出结果 CSV」→ 用 Excel 打开，有「被评人」列；已提交的行有答案与提交时间

### 分享
- [ ] 批次详情「分享给员工」→ 弹出二维码 + 链接 + 复制按钮，链接为 `.../eval-fill`

- [ ] `vite.config.ts` 新增的 proxy 在服务器不生效（由 nginx 处理 /api），无副作用

## 附：常用命令速查

| 目的 | 命令 |
|---|---|
| 查看容器状态 | `docker compose ps` |
| 看后台日志 | `docker compose logs --tail 30 app-backend` |
| 改了后端代码后重启 | `docker compose restart app-backend` |
| 改了 `.env` 后重启（需重载） | `docker compose up -d --force-recreate app-backend` |
| 改了前端代码后重启 | `docker compose restart app-frontend` |

> 提示：Windows 下代码改动不会自动热更新，改完对应服务需手动重启（见上表）。
